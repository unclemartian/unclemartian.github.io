Hexo day-to-day best practise

Posted on 2000-01-01 Edited on 2021-09-14

关于代码管理

  1. hexo blog 管理一切文字 + 图片内容，包括知识，技术和区块链内容
  2. 一律使用 Github Pages + Travis Ci 管理发布
  3. mindmap 使用私人飞书管理，不再参与同步，不上传 hexo
  4. 不再使用 tiddlywiki，因为当文件 > 40MB 时，保存太慢

关于 tags，categories

  1. Title prefix 写大类
  2. 使用 tags 管理细类
  3. 不使用 categories

Rules

  1. 每篇 post 都需要有 prefix，如 【Crypto】，【Tech】
  2. 每个 prefix 都对应一些特定的 tags，比如 nft 对应【Crypto】

举例：

Blog posts：

  • 【Crypto】Auction models 一些拍卖模式介绍
    • tag：nft
    • tag：defi
    • tag：crypto people
  • 【Tech】HTTP learning notes (1/3) all about http
    • tag：ubuntu
    • tag：technology
  • 【Modern Art】Week 2, Art Periods
  • 【读书笔记】明朝那些事2
